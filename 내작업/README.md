# 내작업 폴더

**실습은 전부 이 폴더 안에서 하세요.**

## 쓰는 방법

1. 실습할 파일을 이 폴더로 **복사**합니다. (원본은 그대로 두세요)
2. 복사본을 열어서 자유롭게 코드를 작성하고 실행합니다.
3. 새 자료를 받을 때는 `./업데이트.sh` — 이 폴더는 절대 건드리지 않습니다.

예시:

```
내작업/
└── day02/
    ├── 교안_01.ipynb      ← 복사해서 따라 친 것
    └── 과제_LV1.ipynb     ← 내가 푼 답안
```

## 왜 복사해서 쓰나요?

배포된 원본 파일을 직접 고치면, 다음에 새 자료를 받을 때 충돌이 생겨서
`git pull`이 실패합니다. 노트북 파일은 충돌을 수동으로 푸는 게 매우 어렵습니다.

원본을 깨끗하게 두면 업데이트가 항상 문제없이 됩니다.
