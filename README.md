# 파이썬 실습자료

엔코아 AI캠퍼스 「데이터 분석 & AI 머신러닝 캠프」 실습용 주피터 노트북입니다.
수업 진도에 맞춰 자료가 계속 추가됩니다.

---

## ⚠️ 시작하기 전에 — 딱 하나만 기억하세요

> ## 배포된 실습자료는 **읽기 전용**입니다.
> ## 실습은 **`내작업/` 폴더에 복사해서** 하세요.

`day02_변수_제어문/` 같은 폴더의 파일은 **눈으로 보기만** 하는 원본입니다.
답을 쓰거나 셀을 실행하려면 **먼저 `내작업/` 으로 복사**한 뒤, 복사본을 여세요.

**왜 그런가요?**
원본 파일을 고치면 다음에 새 자료를 받을 때 충돌이 나서 업데이트가 막힙니다.
노트북(`.ipynb`)은 내부가 JSON이라 충돌을 손으로 푸는 게 거의 불가능합니다.
셀을 **실행만 해도** 결과가 파일에 기록되어 "수정됨" 상태가 되니, 원본은 아예 열지 않는 게 안전합니다.

이 규칙 하나만 지키면 학기 내내 아무 문제가 없습니다.

## 1. 준비하기 (처음 한 번만)

### 1-1. 자료 내려받기

```bash
git clone https://github.com/paircodingofficial-cloud/enkoa-practice.git
cd enkoa-practice
```

### 1-2. uv 설치하기

**uv** 는 파이썬 환경과 라이브러리를 관리해 주는 도구입니다. 빠르고 설정이 간단해 이 과정에서 사용합니다.

**macOS / Linux**
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**Windows (PowerShell)**
```powershell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

설치 후 터미널을 **새로 열고** 확인하세요:
```bash
uv --version
```

> `command not found` 가 나오면 터미널을 완전히 껐다 켜 보세요. 그래도 안 되면 강사님께 문의하세요.

### 1-3. 가상환경 만들기

**가상환경**은 이 프로젝트만의 독립된 파이썬 공간입니다.
여기에 설치한 라이브러리는 컴퓨터의 다른 프로젝트에 영향을 주지 않습니다.

저장소 폴더(`enkoa-practice`) 안에서:

```bash
uv venv --python 3.12
```

`.venv` 라는 폴더가 생기면 성공입니다. (이 폴더는 GitHub에 올라가지 않습니다)

### 1-4. 주피터 설치하기

```bash
uv pip install jupyter
```

### 1-5. 주피터 실행하기

```bash
uv run jupyter lab
```

브라우저가 열리면 준비 끝입니다. 종료할 때는 터미널에서 `Ctrl + C` 를 두 번 누르세요.

## 2. 매일 실습하는 방법

### 2-1. 오늘 자료를 `내작업/` 으로 복사

**macOS / Linux**
```bash
mkdir -p 내작업/day02
cp day02_변수_제어문/*.ipynb 내작업/day02/
```

**Windows (PowerShell)**
```powershell
mkdir 내작업\day02
copy day02_변수_제어문\*.ipynb 내작업\day02\
```

> 파일 탐색기에서 끌어다 복사해도 똑같습니다.

### 2-2. 복사본을 열어 실습

주피터 왼쪽 파일 목록에서 **`내작업/day02/`** 안의 파일을 여세요.
`day02_변수_제어문/` 안의 원본은 참고용으로만 보세요.

### 2-3. 정리

| 폴더 | 성격 | 여기서 할 일 |
|---|---|---|
| `day02_변수_제어문/` | 배포 원본 | **보기만** 합니다 |
| `내작업/` | 내 작업 공간 | **여기서 실습**합니다 (GitHub에 안 올라감) |

---

## 3. 새 자료 받기

수업이 진행되면 새 폴더가 추가됩니다. 받을 때는:

```bash
./업데이트.sh
```

처음 한 번은 실행 권한이 필요할 수 있습니다:

```bash
chmod +x 업데이트.sh
```

> **Windows 사용자**는 Git Bash에서 실행하거나, 아래 두 줄을 대신 입력하세요.
> ```bash
> git checkout -- .
> git pull
> ```

이 스크립트는 **`내작업/` 폴더를 절대 건드리지 않습니다.** 배포 원본만 최신으로 맞춥니다.
혹시 원본을 고쳤더라도 `.백업/` 폴더로 옮겨 두니 잃어버리지 않습니다.

| 일차 | 설치 명령 |
|---|---|

> 이후 일차의 라이브러리는 자료가 추가될 때 이 표에 함께 안내됩니다.

## 6. 자주 겪는 문제

**`uv: command not found`**
→ 터미널을 완전히 껐다 켜세요. uv 설치 후에는 새 터미널이 필요합니다.

**주피터에서 커널이 안 잡혀요**
→ `uv run jupyter lab` 으로 실행했는지 확인하세요. `uv run` 없이 실행하면 가상환경이 연결되지 않습니다.

**`./업데이트.sh` 가 실행되지 않아요**
→ `chmod +x 업데이트.sh` 를 한 번 실행하세요. Windows는 위 3번의 두 줄 명령을 쓰세요.

**`git pull` 을 직접 했더니 에러가 나요**
→ `./업데이트.sh` 를 대신 쓰세요. 원본 수정분을 알아서 정리해 줍니다.

**노트북을 실행했더니 파일이 바뀌었다고 나와요**
→ 원본을 직접 실행하신 겁니다. `./업데이트.sh` 가 정리해 주니 괜찮습니다.
   다음부터는 **`내작업/` 에 복사한 뒤** 실행하세요.

**실수로 원본에 답을 써 버렸어요**
→ `./업데이트.sh` 를 실행하면 `.백업/` 폴더에 옮겨 둔 뒤 원본을 되돌립니다. 백업에서 답을 꺼내 `내작업/` 으로 옮기세요.
